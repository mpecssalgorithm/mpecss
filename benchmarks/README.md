# Benchmark Collections Overview

This folder contains benchmark files, metadata, conversion utilities, instructions, and source references for three MPEC/MPCC benchmark suites. These benchmarks are used to evaluate the MPECSS solver against 886 problems from the optimization literature.

---

## What Is an MPEC Benchmark?

An MPEC (Mathematical Program with Equilibrium Constraints) benchmark is a collection of test problems used to evaluate MPEC/MPCC solvers. Each problem is defined by:

- **Decision variables** `x` with bounds `lbx ≤ x ≤ ubx`
- **Objective function** `f(x)` to minimize
- **General constraints** `lbg ≤ g(x) ≤ ubg`
- **Complementarity pairs** `(G_i(x), H_i(x))` with `0 ≤ G_i(x) ⊥ H_i(x) ≥ 0`

The notation `⊥` means `G_i(x) · H_i(x) = 0` — at least one of each complementarity pair must be zero at a feasible point.

---

## Data Format

All three suites use **CasADi JSON** as the primary data format. Each problem is stored as a single JSON file that encodes:

- Problem dimensions (`n_x`, `n_comp`, `n_con`)
- Variable bounds (`lbx`, `ubx`)
- Constraint bounds (`lbg`, `ubg`)
- A CasADi symbolic representation of the objective and constraints (serialized as a computational graph)

The JSON files are loaded at runtime by the `mpecss.helpers.loaders` module:
- `load_mpeclib(path)` for MPECLib `.nl.json` files
- `load_macmpec(path)` for MacMPEC `.nl.json` files
- `load_nosbench(path)` for NOSBENCH `.json` files

The tracked MPECLib JSON files have per-file SHA-256 hashes in `mpeclib/SHA256SUMS.txt`. For the DOI release, record the MacMPEC, NOSBENCH, and official-results archive checksums in the root `CHECKSUMS.md`.

### Suite Characteristics

- **MPECLib**: Classic MPEC problems from the GAMS library. Small to medium sized. Well-studied in the literature — many have known optimal solutions for validation.
- **MacMPEC**: Sven Leyffer's collection of AMPL-format MPEC problems. Medium to large sized. Diverse application domains including bilevel programming and game theory.
- **NOSBENCH**: Problems from nonsmooth optimal control, generated by the `nosnoc` toolbox. Structured, often large-scale. Organized by difficulty level (1–4).

## Downloading the Benchmark Data

### Option 1: Kaggle Dataset (Recommended for Benchmark Runs)

The [official Kaggle dataset](https://www.kaggle.com/datasets/mpecssalgorithm/mpecss-benchmarks) contains all three suites pre-packaged. Attach it as input to a Kaggle notebook.

### Option 2: Download Individual Suites

See each suite's README for download links:
- [MPECLib README](mpeclib/README.md) — JSON files included in repository; GMS source from GAMS World
- [MacMPEC README](macmpec/README.md) — JSON files from SYSCOP Cloud
- [NOSBENCH README](nosbench/README.md) — JSON and MATLAB files from GitHub

### Option 3: GitHub Repository

The MPECLib JSON files are included directly in the GitHub repository:
```bash
git clone https://github.com/mpecssalgorithm/mpecss.git
# mpeclib-json/ is already present
# macmpec-json/ and nosbench-json/ need to be downloaded separately
```

## Running Benchmarks

After downloading the data, run benchmarks using the `resumable_benchmark.py` wrapper:

```bash
# Install the MPECSS package
pip install -e .

# Run MPECLib (92 problems)
python kaggle_setup/resumable_benchmark.py --dataset mpeclib --workers 4 --timeout 3600 --seed 42

# Run MacMPEC (191 problems)
python kaggle_setup/resumable_benchmark.py --dataset macmpec --workers 4 --timeout 3600 --seed 42

# Run NOSBENCH Group 1 (101 problems)
python kaggle_setup/resumable_benchmark.py --dataset nosbench --workers 4 --timeout 3600 --seed 42 \
    --problem-list kaggle_setup/nosbench_splits/nosbench_group1_problems.txt
```

For the full Kaggle workflow, see [`kaggle_setup/README.md`](../kaggle_setup/README.md).
